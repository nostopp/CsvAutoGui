# 校验清单

## 资源清单校验

对 `<config_name>_resource.csv` 检查：

- 每一行都必须是 `resource` 节点
- 只允许 `pic` 和 `ocr` 两种资源类型
- 这个编写清单中不允许出现 `jmp`
- alias 必须唯一
- pic 行必须引用真实存在的图片文件
- pic 和 OCR 行都必须有合法的 `x;y;w;h` 区域
- OCR 行必须有目标文字

## 最终配置校验

对生成出的配置文件检查：

- 被引用的子流程文件真实存在
- 跳转目标或跳转标记真实存在
- 普通流程文件中没有 `resource` 节点
- `recovery.csv` 也是普通流程文件，不能出现 `resource` 节点
- script 路径没有越出配置目录
- 运行时 `*_resource.csv` 只在脚本真的需要时创建
- 如果存在 `runtime.json`，字段名和层级必须符合当前运行时约定
- `recovery_limit` 只能出现在 `watchdog` 下
- 资源命名可理解且带有阶段语义
- 分支逻辑和已确认流程方案一致

## 质量检查

- 优先使用最小但仍然可读的结构
- 当 CSV 分支已经够用时，不要引入脚本
- 当脚本明显能降低脆弱性时，也不要强行把逻辑塞进 CSV
- 优先复用，而不是复制粘贴后逐渐漂移

## 面向用户的最终提醒

如果这次改了文件，提醒用户运行时缓存不会自动刷新。
明确告诉用户在运行新配置前需要使用界面里的“重新加载”。
