# 项目模板

## 适用范围

这个文件用来记录项目特定、稳定且可复用的配置编写知识。
不要把它写成宽泛的业务说明文档。
只记录已经验证的业务规律，不复制框架模块路径、CSV 契约或 `runtime.json` 结构；这些内容以共享 reference 和当前源码为准。

## 证据来源

- 这些规律来自哪些已经验证的目标配置。
- 最近一次通过真实 UI 验证的时间或版本。
- 仍然存在的适用条件或不确定性。

## 骨架选择

- 哪些现有配置是主要参考骨架。
- 各骨架分别适合什么情况。

## 常见阶段

- 典型起始状态。
- 典型主要阶段。
- 典型恢复阶段。

## 常见锚点

- 稳定的 OCR 锚点。
- 稳定的 pic 锚点。
- 哪些情况更适合用 OCR，哪些情况更适合用 pic。

## 分支模式

- 反复出现的异常分支。
- 它们通常在什么阶段出现。
- 处理完成后通常回到哪里。

## 命名约定

- alias 的命名习惯。
- 流程文件的命名习惯。
- 子流程如何拆分的约定。

## 已知坑点

- 值得记录的高价值错误或脆弱状态。
