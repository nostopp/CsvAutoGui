# CsvAutoGui 配置编写

## 目标

把用户确认过的自动化逻辑翻译成符合当前仓库契约的配置，并在交付前给出可复现的校验结果。

不要根据模糊描述发明未知 UI 状态、截图、OCR 锚点或业务策略。信息不足时先整理已知流程，只追问会改变实现或阻止验证的问题。

## 选择任务模式

开始时先判断任务属于哪一种模式；一个任务可以组合多个模式。

| 模式 | 适用情况 | 默认路径 |
| --- | --- | --- |
| 创建 | 新建完整配置 | 五阶段全部执行 |
| 修改 | 扩展或重构已有配置 | 先检查现状，只补受影响阶段和新增资源 |
| 审查 | 检查配置但不要求修改 | 保持只读，直接执行流程确认和最终校验 |
| 运行策略 | 只调整 `runtime.json`、watchdog 阈值或通知策略 | 跳过无关的资源阶段 |
| 脚本扩展 | 新增或修改 `script` 与运行时资源 | 先证明 CSV 不适合，再检查脚本契约 |

不要因为 skill 支持完整工作流，就强迫简单审查或运行参数调整重新采集资源。

## 读取参考资料

- 始终完整读取 [references/core.md](references/core.md)。
- 所有模式都完整读取 [references/workflow.md](references/workflow.md)，只执行与当前模式相关的章节。
- 只读审查在 workflow 之后立即读 [references/validation.md](references/validation.md)；其他模式在最终写文件前必须读它。
- 只有涉及对应能力时才读 [references/patterns.md](references/patterns.md) 中的分支、脚本、watchdog、recovery 或通知示例。
- 目标项目族在 `references/projects-local/` 中有对应文件时，只读取该项目文件。

共享参考资料之后才读取项目本地知识。项目本地知识只能补充业务实现规律，不能覆盖当前框架契约。

## 五阶段门禁

### 1. 确认流程

输出一份紧凑的流程规格，至少包含状态或阶段、转移条件、资源、预期文件、未决问题，以及使用 CSV、script、watchdog 和 recovery 的理由。

用户已经给出完整且明确的规格并要求实施时，可视为本阶段已确认。否则，在用户确认前不要写目标配置。

### 2. 准备资源

仅当实现需要新增或替换 `pic/ocr` 锚点时进入本阶段。生成资源采集清单并说明每一项的用途、采集状态和目标阶段；不要把已有且已验证的资源重新列为待采集。

### 3. 校验资源

资源不完整、路径越界、alias 冲突或目标文件不存在时停止生成依赖这些资源的最终流程。输出逐项通过/失败报告，不自行猜测缺失内容。

### 4. 生成或修改配置

只写用户选定的目标 config。保持改动最小，列出创建、修改和建议清理的文件；不要把其他 config 的旧导入或运行时结构复制进目标配置。

### 5. 校验最终输出

运行 [scripts/validate_config.py](scripts/validate_config.py)，再按 [references/validation.md](references/validation.md) 做任务特定复核。任何错误或警告都必须解决或明确报告，不能用“看起来正确”代替执行结果。

## 编写清单生命周期

`<config_name>_resource.csv` 使用了框架保留的运行时资源后缀，但在阶段 2、3 中可以暂时作为编写清单。它只能包含 `操作=resource` 且 `操作参数=pic;alias` 或 `ocr;alias` 的节点，不能包含 `jmp;alias`。

进入最终交付前必须明确处理它：

1. 脚本确实需要时，将它有意识地转为脚本引用的运行时资源文件；
2. 已把资源落入普通流程且清单不再需要时，征得用户同意后删除；未获删除授权时明确列为阻止最终校验通过的待清理文件；
3. 尚未完成资源采集时保留它，但不得宣称最终配置已经完成。

不要无说明地把未被任何 script 使用的 `*_resource.csv` 留在最终配置中，也不要未经授权删除用户已有文件或素材。

## 写入边界

- 可以读取其他 config 作为业务骨架参考，但只修改选定目标 config 和本 skill 自身明确要求的文件。
- 不修改作为参考来源的 config，不覆盖或删除用户图片，不生成越出目标 config 的路径。
- `references/projects-local/` 默认只提出候选更新；用户明确要求维护项目知识时才写入。
- 审查模式保持只读，除非用户随后明确要求修复。

## 完成标准

最终回复必须包含：

- 实际创建、修改和建议清理的文件；
- 执行过的校验命令及结果；
- 尚未验证的真实 UI 行为或外部依赖；
- 发生 CSV、`recovery.csv`、`runtime.json`、脚本或运行时资源改动时，提醒用户在运行前手动“重新加载”。
